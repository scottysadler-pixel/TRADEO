# Trade1 — User guide

This guide describes **what the project does today**, how to **use the Trade app**, how to **refresh data**, and how to **prove the app is loading real inputs** (or know when it is stale).

---

## 1. What you have now

### A. Trade app (main product)

**Files:** `standalone/index.html` and `standalone/trade.html` (identical).

**Features (tabs):**

- **Signals** — Two strategies:
  - **The Chameleon** — switches between trend-following (calm volatility) and fading extremes (high volatility).
  - **Catchup Trader** — uses **5-day** gold vs AUD momentum divergence (commodity linkage).
- **Position size** — rough micro-lot sizing from balance, risk %, and stop pips (illustrative).
- **Journal** — optional trade log in **your browser** (`localStorage`; not sent anywhere).
- **Learn** — short definitions and broker ideas (not endorsements).
- **Data health** — timestamps baked into `chameleon_data.json` / `catchup_data.json`, age warnings, and an optional **live AUD/USD check** against the public Frankfurter API (when your browser allows it).

**Data files the app reads (same folder):**

- `chameleon_data.json` — built by `scripts/update_chameleon.py`
- `catchup_data.json` — built by `scripts/update_catchup.py`

Those scripts use **`data/prices.csv`**, **`data/rates.csv`**, **`data/commodities.csv`** on your machine (or in CI). If you never refresh, the JSON (and signals) stay old.

### B. Analyst trial bundle (optional / advanced)

**File:** `standalone/trial_dashboard.html` (generated by `npm run trial`).

Full multi-preset backtest UI, replay, health JSON, etc. **Not** the same as the Trade app. The build **does not overwrite** `index.html` anymore.

## 2. Is the data “real time”? Is it “realistic” for trading?

**No live streaming.** The Trade app is **static HTML + JSON**. Numbers are only as fresh as the **last time** someone ran the update scripts (or CI).

- **AUD/USD** in the pipeline is typically **daily closes** (e.g. Frankfurter / ECB-style business-day series), not tick data.
- **Gold** comes from public market data (e.g. Yahoo via Python) aligned to your price calendar.
- **Signals** are **rules on that history** — useful for learning and paper workflow, **not** a guarantee of profit.

**Realistic use:**

- Treat signals as **slow, daily context**, not scalping entries.
- Always compare the shown AUD/USD to a **live quote** from your broker or a finance site before any real order.
- **Demo account first** until you trust the refresh rhythm and the Health tab.

## 3. Quick test — “Is it connected?”

Do these in order.

### Step 1 — JSON loads (proves hosting works)

1. Open the Trade app over **http/https** (not `file://`).
2. Open **Data health**.
3. You should see **Last bundle update** times for Chameleon and Catchup (from JSON). If it says **failed to load JSON**, the server path is wrong or files are missing from `standalone/`.

### Step 2 — Timestamps are recent (proves you refreshed)

After a local refresh (see §5), **Last bundle update** should be within minutes. If it is **days old**, the app is still “working” but **stale** — run the refresh commands again or redeploy.

### Step 3 — Live quote check (proves the internet path)

On **Data health**, click **Run live quote check**.

- **OK:** Frankfurter returned a rate; compare to the **AUD/USD** shown under Signals (same ballpark; small gaps are normal).
- **Failed / blocked:** Some networks or browsers block cross-origin calls. Use Step 4 instead.

### Step 4 — Manual cross-check (always works)

1. Note **AUD/USD** on the Signals tab.
2. Open [Frankfurter latest AUD→USD](https://api.frankfurter.app/latest?from=AUD&to=USD) or any trusted quote page.
3. Compare — should be close if your `prices.csv` / JSON were refreshed recently.

### Step 5 — Regenerate JSON and reloadFrom repo root:

```powershell
python scripts/update_chameleon.py
python scripts/update_catchup.py
```

Hard-refresh the browser (Ctrl+F5). **Data health** should show new times.

## 5. Refreshing data on your PC (full pipeline sketch)

Minimal path to refresh **inputs** used by the two strategies:

```powershell
npm run fetch:price
python aud_strategy/scripts/fetch_real_data.py
python scripts/update_chameleon.py
python scripts/update_catchup.py
```

(`fetch_real_data.py` needs `data/prices.csv`; `npm run fetch:price` creates it. Optional: `FRED_API_KEY` in `.env` for Fed series — see `aud_strategy/README.md`.)

Then redeploy or copy `standalone/*.html` and `*.json` to your host.

**Convenience:** `npm run refresh:apps` runs only the two update scripts (expects CSVs already present).

## 6. GitHub Actions / Pages (optional)

Workflows may build and deploy **`standalone/`** to GitHub Pages. They are **optional**; many users rely on **Vercel + local refresh** only. If Actions fail, your local Trade app and Vercel deploy from git are still valid as long as committed JSON/HTML are fresh.

## 7. Troubleshooting

| Symptom | Likely cause | Fix |
|--------|----------------|-----|
| Blank signals or errors | Opened HTML as `file://` | Use `python -m http.server` in `standalone/` or Vercel |
| Old “Trial dashboard” on `/` | Old `index.html` in git | Ensure git `standalone/index.html` is the Trade app; redeploy |
| Live check always fails | CORS / network | Use §3 Step 4 manual check |
| Numbers don’t match broker | Daily close vs live bid/ask | Expected; use broker price for orders |
